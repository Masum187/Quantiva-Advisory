# How-to: Case anlegen

Dieser Guide zeigt, wie du einen neuen **Case** im Admin-Dashboard erstellst.

## Schritte

1. **Neuer Case** → Button in der Topbar klicken.
2. **Slug** vergeben (nur `a-z0-9-`), z. B. `btp-delivery`.
3. **Titel (DE/EN)**, optional **Untertitel** eintragen.
4. **Kategorie** & **Branche** wählen.
5. Optional **Hero Media** (Bild/Video Pfad) setzen:
   - Bild: `/assets/cases/<slug>-hero.jpg`
   - Video: `/assets/cases/<slug>-hero.mp4`
6. **Ziele/Lösung/Ergebnisse** als kommagetrennte Listen eingeben.
7. **Tech** (Tags) kommagetrennt.
8. Optional **Zitat** (DE/EN + Autor).
9. **Speichern**.

> **Validierung:** Live-Fehler erscheinen direkt im Drawer (z. B. ungültiger Slug, fehlender Titel).

## Bulk-Tipps

- In der Liste mehrere Zeilen markieren → **Bulk-Leiste** nutzen, um
  - Kategorie/Branche **zuweisen/entfernen**
  - **Tech** tags **hinzufügen/entfernen**
  - **Clear**-Aktionen (z. B. Kategorie leeren)

## Import/Export

- **Export JSON**: sichert alle Cases als `cases.json`.
- **Import JSON**: lädt Cases (validiert; Warnung bei fehlerhaften Einträgen).

## Undo/Redo & History

- Jeder Speichervorgang landet in der **History** (gebatcht, ~300 ms).
- **Undo/Redo** knöpfe + **History Panel** mit Snapshots, Vorschau & Wiederherstellen.
