# Content Model

Das Content-Modell beschreibt die Entities und deren Relationen, die im CMS verwendet werden.

## Entities

### Case

- `slug: string` *(unique)*
- `titleDe/titleEn: string?`
- `subtitleDe/subtitleEn: string?`
- `category: "Cloud" | "Data" | "Integration" | "Security" | "Quality" | "Enablement" | null`
- `industry: "Pharma" | "Healthcare" | "Logistics" | "Manufacturing" | "Retail" | "Finance" | "Public" | null`
- `heroImage: string?` (Pfad, z. B. `/assets/cases/slug-hero.jpg`)
- `heroMedia: string?` (Pfad, z. B. `/assets/cases/slug-hero.mp4`)
- `goalsDe/goalsEn: string[]`
- `solutionDe/solutionEn: string[]`
- `resultsDe/resultsEn: string[]`
- `tech: string[]`
- `quote?: { textDe?: string; textEn?: string; author?: string }`
- `status?: "draft" | "inReview" | "approved" | "rejected" | "published"`
- `owner?: string | null`
- `reviewers?: string[]`
- `publishedAt?: string | null`

## Taxonomy

- **Kategorien**: Cloud, Data, Integration, Security, Quality, Enablement  
- **Branchen**: Pharma, Healthcare, Logistics, Manufacturing, Retail, Finance, Public  
- **Tech**: freie Liste (Tags)

## Medien

- Bilder: `/assets/...` (`.jpg/.jpeg/.png/.webp`)
- Videos: `/assets/...` (`.mp4/.webm`)
- Naming-Konvention: `/<area>/<type>/<slug>-<role>.<ext>` (z. B. `/assets/cases/btp-delivery-hero.jpg`)

## Richtlinien

- **Titel**: prägnant, nutzenorientiert  
- **Untertitel**: 1-2 Sätze Kontext  
- **Goals/Solutions/Results**: Listen, je 1–5 Items  
- **Quote**: nur, wenn vorhanden; mit Autor  
- **Barrierefreiheit**: Alt-Texte für alle Bilder

## Roadmap (optional)

- Relationen (z. B. Case ↔ Referenzen/Autoren)
- Lokalisierung (mehr Sprachen)
- Versionierung & Diff
- Headless-API (REST/GraphQL)
